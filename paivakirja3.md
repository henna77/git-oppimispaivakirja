# Oppimispäiväkirja: Git projektissa

__Mitä hyötyä voisi olla versionhallinnasta, jos kehität projektia yksin?__

Versionhallinta antaa varmuuskopiot jokaisesta muutoksesta. Jos rikot jotain, voit palata aiempaan commit-versioon ilman paniikkikohtausta.
Voin työskennellä useammalla koneella ilman usb-tikkujen, sähköpostien tai zip-pakettien kanssa sekoilua.

__Mitä hyötyä voisi olla versionhallinnasta, jos projektissa on useita kehittäjiä?__

Kaikki pääsevät tekemään töitä yhtä aikaa.
GitHub näyttää muutokset selkeästi: kuka teki mitä, milloin ja miksi.
Tiimi voi keskustella pull requestien kautta ennen kuin mitään yhdistetään.

__Miten järjestäisit projektitiimin versionhallinnan 3-4 hengen ohjelmistoprojektikurssilla? Laadi tiimiläisille lyhyt ohje, miten projektissa toimitaan.__

1. Päähaara sisältää vain toimivan version.
    - Aina kun koodi yhdistetään mainiin, sen täytyy olla testattu ja toimiva.
2. Kehityshaara on varsinainen työskentelyhaara.
    - Kaikki ominaisuudet yhdistetään ensin developiin.
3. Jokainen ominaisuus tehdään omassa haarassa
4. Vähintään yksi tiimikaveri hyväksyy PR:n ennen mergeä.
    - Tämä estää virheet ja pakottaa katsomaan muutokset läpi
5. Palaveri 1–2 kertaa viikossa.
    - Sovitaan seuraavat tehtävät.
__Kommenttini opintojaksosta, esim. sisällöstä, materiaalista, työmäärästä, hyödyllisyydestä, työmäärästä. Mitä toivoisit olevan enemmän, mitä vähemmän?__
